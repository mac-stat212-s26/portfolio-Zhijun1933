Data Story Critique

Source: How life has changed for people your age — ABC News Australia

What is the data story? The piece explores how key life metrics — housing, income, education, family structure, and employment — have shifted across generations in Australia. Readers can input their age to receive personalized generational comparisons.

What is effective? The interactive, personalized design is highly engaging: readers see data relevant to their own generation rather than abstract trends. The visualizations are clean, and the cross-temporal comparisons are intuitive and accessible to a general audience.

What could be improved? The story could better contextualize the structural drivers behind the observed changes (e.g., policy shifts, macroeconomic cycles). Some charts would benefit from confidence intervals or sample size indicators to convey uncertainty. The interactivity, while compelling, may also obscure the broader systemic narrative in favor of individual-level framing.

Exercise 3: Counting by cut — == vs %in%

How many diamonds are of Fair, Premium, or Ideal cut? What fraction of the total?

Wrong way with ==:

Using == against a length-3 vector triggers vector recycling. Because 1000 (the number of rows) is not a multiple of 3, R raises a warning: “longer object length is not a multiple of shorter object length.” Beyond the warning, the result is incorrect — R compares element-wise with recycling rather than checking set membership.

Code
diamonds |> 
    filter(cut == c("Fair", "Premium", "Ideal")) |> 
    nrow()
[1] 226

Correct way with %in%:

Code
# Total count
diamonds |> 
    filter(cut %in% c("Fair", "Premium", "Ideal")) |> 
    nrow()
[1] 685
Code
# Fraction
mean(diamonds$cut %in% c("Fair", "Premium", "Ideal"))
[1] 0.685

Exercise 4: any() and all()

# Any Fair cut diamonds > $3000?
fair_prices <- diamonds |> 
    filter(cut == "Fair") |> 
    pull(price)
any(fair_prices > 3000)
[1] FALSE
Code
# Are ALL Ideal cut diamonds > $2000?
ideal_prices <- diamonds |> 
    filter(cut == "Ideal") |> 
    pull(price)
all(ideal_prices > 2000)
[1] FALSE

Exercise 5: if_else() and case_when()

Create two categorized versions of price.

Code
diamonds <- diamonds |> 
    mutate(
        price_cat1 = if_else(price < 500, "low", "high"),
        price_cat2 = case_when(
            price < 500 ~ "low",
            price >= 500 & price <= 1000 ~ "medium",
            .default = "high"
        )
    )

diamonds |> 
    select(price, price_cat1, price_cat2)
