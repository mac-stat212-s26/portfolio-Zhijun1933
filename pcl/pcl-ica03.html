Pre-class learning map

R4DS Chapter 1 walks through ggplot2 not as a list of geoms but as a grammar — a small number of orthogonal pieces that compose. The chapter’s central pedagogical move is showing that almost every plot you’ll ever make is data + aesthetic mapping + geom + (sometimes) facets + (sometimes) labels, with each piece making one independent decision. The mindmap below preserves that orthogonality rather than flattening it into a list of “here are the 30 geoms.”

Code 
mindmap
  root((R4DS Ch 1 ggplot grammar))
    Three irreducible inputs
      data
        a data frame · one row per observation
      aesthetic mapping
        which variable maps to which channel
        x · y · color · shape · size · alpha
        "aes() lives in ggplot or in geom"
      geometric object
        what visual mark is drawn
        geom_point · geom_line · geom_bar
        geom_smooth · geom_boxplot
        layers stack with +
    Variable types drive choices
      categorical
        bar · count · fill
        boxplot for one cat × one num
      numerical
        histogram · for one num
        scatter · for two num
        smooth · for trend over scatter
      ordering matters
        fct_reorder · sort bars by value
        fct_infreq · sort by count
    Multi-panel plots
      facet_wrap · ~var
        wraps panels in a grid
      facet_grid · row ~ col
        full crossing of two variables
      shared scales by default
        scales = free to release
    Labels and styling
      "labs(title, x, y, color, caption)"
      theme_minimal · _bw · _classic
      "theme() to override specifics"
    Saving
      "ggsave()"
        last printed plot by default
        path · width · height · dpi
        pick raster · png · or vector · pdf
    Common pitfalls
      "aes() vs setting outside aes()"
        color = species in aes · maps a variable
        color = "blue" outside aes · sets a constant
      + at end of line not start
      geom inherits ggplot aesthetics
        unless overridden in the geom
      missing values
        warned, then dropped

Three things the chapter wants you to internalize

The grammar is compositional, not enumerative. A scatterplot colored by species with a smoothed trend line is ggplot(data, aes(x, y, color = species)) + geom_point() + geom_smooth(). Three independent decisions: (1) what data, (2) what mapping, (3) what marks. Memorizing “the scatter-with-trend recipe” misses the point — once you see the grammar, you can build any plot by deciding each piece independently. This is why the chapter spends pages on aes() before introducing more than a couple of geoms.

aes() vs constants is the single most-confused detail. geom_point(aes(color = species)) says “let species drive the color”; geom_point(color = "blue") says “make every point blue.” Putting color = "blue" inside aes() gets you a confusing legend with one entry called “blue,” because ggplot tries to treat the literal string "blue" as a variable level. This trips up almost everyone the first time.

Inheritance flows from ggplot() down to geoms. Aesthetics declared in ggplot(data, aes(x, y)) are inherited by every geom layered on top, so subsequent geoms don’t need to repeat them. Per-geom aes() blocks override or extend the global mapping. This is what makes ggplot(data, aes(x, y)) + geom_point() + geom_smooth() work without reciting aes(x, y) twice.

Why “advanced” data viz starts here

ICA 03 is labeled Advanced Data Viz, but the prep is R4DS Chapter 1 — the introductory chapter. The reason isn’t that the course is starting from scratch; it’s that advanced visualization in this course means using the grammar deliberately rather than instinctively. The exercises and TidyTuesdays later in the semester will ask for things like “recreate this NYT chart with a custom theme and an annotation layer that draws attention to a specific subset” — which only works if the grammar’s joints are already familiar enough to recombine without thinking. The chapter is review for someone who has used ggplot before, but it’s review with the explicit goal of making the joints visible.
